## 本轮最主要的发现：Module 9 倾向于重复 Module 1 已经用过的头条发现

样本1、7、8 都出现同一个模式：Module 1（关系运行地图）选中的"C节拍"headline，在对应的专题模块（Module 3 或 Module 6）里被完整展开一次，然后 **Module 9 的候选池又把同一个模式选出来，用高度相似的措辞第三次陈述**。

例如样本7/8："关系信号变模糊时先减少投入保护自己 + 还没吵起来先压下去 = 对外不可见的保护链"这个发现：
- Module 1 提了一次（"这两步合在一起，指向一条对外几乎看不见的自我保护链"）
- Module 6（保护策略链）完整展开了一次
- Module 9 又用几乎相同的句式重述了一次（"这两步方向一致，共同构成一条对外几乎不可见的保护链"）

**根因**：现有规则里，Module 1 的 C 节拍有"去重"要求（不能和其他节拍重复），Module 9 内部候选之间也有"去重合并同根源张力"的要求——但**没有一条规则要求 Module 9 的候选池排除掉 Module 1 已经用过的 headline**。Module 9 的候选池定义是"全卷所有触发的模式"，天然会重新捞到 Module 1 已经讲过的那个最强信号。

样本2/3/4/5/6 没有出现三次重复（只有 Module 1 + 专题模块的两次重复，这是允许的承接，不算违规），说明这不是必然发生，但8组里3组命中，**发生率不低，值得在下一轮批量测试前先补一条规则**。

---

## 建议（不在本次测试中执行，留给你决定）

1. **给 Module 9 的候选池加一条排除规则**：候选池 = 全卷触发的模式 − Module 1 已经作为 C/D/E 节拍标题句用过的那个模式。Module 9 应该呈现"报告至今还没说过的"次强信号，而不是重复最强信号。
2. F选项+空文本的边界情况这一轮没有测到（8组里只测了1组F+真实文本），下一批需要补上。
3. 这轮样本量还不足以对"张力误判""杠杆点错误"这类低频问题下结论——0次不代表不存在，可能只是这8组没有踩中，扩大样本量时优先保留能触发极端/边界组合的设计。

## 与第一轮压测的方法论对比

第一轮（`PROMPT_STRESS_TEST_V1.md`）由我扮演模型角色生成，测的更多是"规则本身写得对不对"；这一轮由全新、不知道我们讨论过程的 agent 独立执行，测的是"规则在没有额外背景知识的情况下能不能被正确执行"——**这一轮的干净程度（尤其是六大禁止语义族0命中）比第一轮更有说服力**，因为它排除了"我自己太熟悉规则所以自然不会犯错"这个混杂变量。
