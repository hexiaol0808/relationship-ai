# 报告分析引擎骨架（7 核心变量）

对应 Phase 1「定稿报告10大模块结构」任务。本文档是 15 题原始答案 → 报告内容之间的推断层规范（该推断什么、该说什么内容），供未来写 AI Prompt 模板时直接引用。

**这些内容该用什么语气、什么强度、什么句式写出来，不在本文档定义，统一写在 `REPORT_TONE_GUIDELINES.md`**——本文档里出现的"输出结论格式"字段只是内容结构模板，最终措辞必须过一遍语气规范文档，尤其是证据强度分级和措辞三档词库。

---

## 单侧数据硬规则（写入系统 Prompt，不是免责声明）

问卷只采集用户单方面的自我报告，没有伴侣数据。dyadic 关系研究反复证明，双方数据存在相互依赖性（actor effect / partner effect 无法从单方回答里拆分），因此：

**Rule 1** 只能分析用户自身的需要、感受、倾向和策略，不得分析或描述伴侣。

**Rule 2** 所有涉及"互动循环"的表述必须使用条件推演句式："如果伴侣也倾向于……，你们之间可能形成……"，禁止断言式写法（如"你们就是追逃型关系"）。

**Rule 3** 不得通过用户的回答反推、诊断伴侣本人（禁止出现"TA是回避型""TA缺乏共情能力""TA控制欲强"这类表述）。

**Rule 4** 报告第一次出现互动循环推演时，必须显式声明："以下关于互动循环的内容，是根据你的回答推演出的可能关系情境，用于帮助你识别模式，并不是对伴侣本人或真实双方关系的诊断。"——这句话要作为系统 Prompt 的固定指令，而不是页面角落的一行小字免责声明。

## 7 核心变量与 15 题的归属关系

**原则：一道题可以参与多个变量的交叉验证，但只能有一个"主归属变量"（primary ownership）。** 这样保证 15 题的信息不会因为反复被不同变量使用而互相稀释或者重复计分。

| 变量代码 | 变量名 | 主数据（primary，唯一归属） | 交叉验证（secondary，可多个变量共用） |
|---|---|---|---|
| V1 | 关系需要与爱的翻译系统 | CP1, CP2, CP3, S6 | — |
| V2 | 急性情绪触发点 | S5 | CP3（借用 V1 的数据） |
| V3 | 威胁下的保护策略链 | S2, S7, S3 | G8, S1（借用 V4 的数据） |
| V4 | 连接—安全状态画像 | G8, S1, S9, G7 | S2（借用 V3 的数据） |
| V5 | 核心修复路径 | G6 | S3（借用 V3 的数据） |
| V6 | 发展变化中的关系协调模式 | G2 | G7（借用 V4）、CP3（借用 V1） |
| V7 | 主观成长目标 | G1 | 全卷（前六个变量的综合诊断） |

15 题逐一核对：CP1→V1，CP2→V1，CP3→V1，S6→V1，S5→V2，G7→V4，G8→V4，S1→V4，S2→V3，S9→V4，S3→V3，S7→V3，G6→V5，G2→V6，G1→V7。共15，每题恰好一个主归属，无遗漏、无重复。

## 变量定义（六字段规范）

### V1｜关系需要与爱的翻译系统

**输入题目**：CP1（选二·接收偏好）、CP2（选二·表达偏好）、CP3（两步·核心情感需要，含F）、S6（单选·错位应对，含F）

**主推断**：
- CP1 第一/第二优先 → 接收爱的频道排序
- CP2 第一/第二优先 → 表达爱的频道排序
- CP3 第一步（+第二步）→ 长期最容易被消耗的核心需要（+ 相对更能适应的次要需要）
- S6 → 面对"爱没对上频道"时的应对策略

**交叉验证**：CP1 vs CP2 一致/不一致（Complementary Dimensions，不判矛盾）；CP3 与 S5（V2）对照，判断长期需要与急性触发是否指向同一机制。

**冲突条件（触发"内部张力"叙事的情况）**：
- CP1 首选 ≠ CP2 首选 → "你最想被接住的方式和你自己最自然会做的却不同"
- CP3 揭示的核心需要方向与 S6 选择的应对策略方向不符（例：CP3=被承接，S6=先观察TA能否自己意识到）→ "你最需要的是被主动照顾，但你处理错位的方式却是被动等待对方想通"

**禁止推断**：不得推断伴侣真实 Love Language；不得推断"当前关系一定有问题"；不得暗示 S6 某个选项更成熟（C 有显式代价句不代表 C 代价更大，A/B/D/E 只是文案精简）；不得把 CP1≠CP2 写成"关系错位"。

**输出结论格式**：
> "你最想被爱的方式是[CP1-1st]，其次是[CP1-2nd]；你自己最自然的表达方式是[CP2-1st]……当这种爱没有对上频道时，你的第一反应通常是[S6]。"

### V2｜急性情绪触发点

**输入题目**：S5（单选+F·急性情绪触发）

**主推断**：S5 选项 → 触发类型（情绪被否定 / 承诺被违背 / 冲突中被回避 / 言语攻击 / 边界被越过）

**交叉验证**：CP3（V1）→ 判断急性触发与长期核心需要是否同源

**冲突条件**：CP3 对应的机制与 S5 对应的机制不一致时 → Level 3 情境对照叙事

**禁止推断**：不得说"矛盾"或"你以为需要A其实需要B"；不得从单题跳到人格结论或依恋类型

**输出结论格式**：
> "最容易让你当场情绪爆发的是[S5]。这和你长期最缺的部分[一致，是你最核心、最一以贯之的敏感点 / 不完全相同——长期滋养关系的东西和当下最容易点燃情绪的瞬间，可能并不是同一件事]。"

---

### V3｜威胁下的保护策略链

**输入题目**：S2（单选+F·模糊信号反应）、S7（单选+F·冲突前不满处理）、S3（单选+F·冲突启动反应）

**主推断**：三题分别对应"感觉不对劲（威胁模糊阶段）→ 不满积累（冲突前阶段）→ 冲突爆发（启动阶段）"这条连续时间线上的策略。

**交叉验证**：G8、S1（V4）→ 判断这条策略链是否随信任程度或关系稳定预期而变化。

**冲突条件**：
- S2=减少投入 + S7=先压下去 组合 → "你倾向于安静地自我保护，很少主动确认，这条保护链对外界几乎不可见"
- S7 与 S3 方向不一致（如 S7=及时表达但 S3=先冷静独处）→ Contextual Contrast，不判矛盾

**禁止推断**：不得把 S3 任何选择解释为"回避型人格""情绪化人格"；不得把 S7 的间接表达解释为"不成熟""不会沟通"；不得把 S2 的策略选择解释为依恋类型。

**输出结论格式**：
> "当关系信号变得模糊时，你的第一反应是[S2]；如果不满还没发展成正式冲突，你通常[S7]；一旦冲突真正开始，你最先想做的是[S3]。这三步连起来，是你在关系张力升高时的完整保护路径。"

---

### V4｜连接—安全状态画像（旗舰变量）

**输入题目**：G8（单选·信任加深行为标记）、S1（单选·稳定基线）、S9（单选+F·忙碌容忍）、G7（选二+F·现实压力容忍）

**主推断**：四题分别对应"信任加深时 → 关系稳定的理想状态 → 忙碌现实状态 → 巨变压力状态"四种情境下的连接需求与行为变化。

**交叉验证**：S2（V3）→ 当连接出现异常信号（威胁触发）时，这套画像是保持一致还是剧烈改变。

**冲突条件**：
- S1 显示高密度需求但 G7 显示对多种变化接受度很高 → "理想中偏好紧密关系，但现实适应力其实不差"
- S1 显示偏独立但 S9 显示忙碌时仍需要高频连接 → 不判矛盾，而是"你不需要大量共同生活，但需要一条稳定的连接线始终存在"

**禁止推断**：不得推断"选择独立/自主类选项=更成熟"或反之；不得推断 S1 与当前关系实际相处频率一致；不得把这几题的情境切换写成"你说一套做一套"；不得把 G8 解读为"依恋类型改变"；不得把 G7 的选择解读为"不够重视关系"。

**输出结论格式**：
> "当你越来越信任一个人时，你会[G8]；在关系稳定、没有压力的日常里，你最舒服的状态是[S1]；即使双方都很忙，你最低需要[S9]；如果生活发生异地、搬家这类巨大变化，你相对最能接受失去的是[G7-1st]和[G7-2nd]。"

**核心金句模板（用户提议，采纳）**：
> "你不是简单地'黏人'或'独立'。你在稳定状态下___，压力状态下___，而真正让你的保护系统启动的是___。"

---

### V5｜核心修复路径

**输入题目**：G6（单选+F·修复优先序）

**主推断**：G6 选项 → 哪种修复成分（理解优先 / 责任承担优先 / 行为改变优先 / 亲密感优先 / 解决方案优先）最先让用户放下防御。

**交叉验证**：S3（V3）→ 冲突刚开始时的本能反应，与 G6 对照，判断"本能策略是否通向真实修复需要"。

**冲突条件**：S3 与 G6 方向不一致（例：S3=先讲清楚/问题导向，G6=先恢复亲密感）→ Contextual Contrast，"你在冲突刚发生时可能本能地想尽快把事情讲清楚，但真正让修复生效的，却是先感受到彼此还站在一起——'想赶快处理'和'真正被安抚住'之间可能存在时间差。"

**禁止推断**：不得把 G6 选择"先恢复彼此的亲密感"解释为"缺乏安全感"；不得跳到人格结论。

**输出结论格式**：
> "一场冲突之后，真正能让你开始放下防御、重新靠近对方的是[G6]。[若与S3方向不一致，补充时间差叙事]"

---

### V6｜发展变化中的关系协调模式

**输入题目**：G2（单选+F·伴侣发展机会应对）

**主推断**：G2 选项 → 面对伴侣的正向重大变化，用户最优先希望关系怎样应对（庆祝优先 / 共同规划 / 现实评估 / 未来位置确认 / 自主空间优先）。

**交叉验证**：G7（V4）→ 判断"正向变化"应对模式和"中性/负向变化"容忍度是否一致；CP3（V1）→ 判断 G2 的选择是否呼应用户最深层的核心需要。

**冲突条件**：G2 揭示的应对方式与 CP3 揭示的核心需要方向不一致时，可用 Level 4 谨慎措辞提示。

**禁止推断**：不得把 G2 的选择做道德评价（例如把 E"给TA充分自主空间"解读为"不够关心伴侣"——这是错误推断，E 只是自主偏好，不是冷漠）。

**输出结论格式**：
> "当TA迎来重要的发展机会时，你最优先希望发生的是[G2]。[若与CP3呼应：这和你在关系里最深层的需要——[CP3]——是一致的，说明即使是好事发生，你也会本能地用同一种方式确认自己在这段关系里的位置。]"

### V7｜主观成长目标（收尾变量）

**输入题目**：G1（单选+F·想提升的关系能力）

**主推断**：G1 选项 → 用户主观意愿层面最想练习的方向。

**交叉验证**：全卷（V1-V6 的综合诊断结果）→ 判断 G1 的主观意愿，是否恰好对应前六个变量识别出的"最高杠杆改变点"。

**冲突条件**：G1 选择与全卷综合诊断的"最高杠杆点"是一致、不一致但可补充、还是互补——三选一判断（判断标准见下方 Module 10 的三类对照表）。这是全份报告收尾模块最重要的判断时刻。

**禁止推断**：不得把 G1 等同于"用户当前最欠缺这项能力"；不得用否定用户自主判断的语气（禁止"你应该练别的，不是这个"）；不得把"互补"关系误判成"不一致"（例如"理解TA"和"被TA理解"不是替代关系，不能写成"除了这个，还有另一个更该练的"，而要写成"这是同一件事的两个侧面"）。

**输出结论格式**：
> "你目前最想练习的是[G1]。综合全卷证据，这和我们从你的回答里看到的最主要模式[是 / 不完全是]同一个方向——……"

## Module 1｜关系运行地图摘要：生成规则

**要解决的问题**：如何从7个变量压缩成3-5条结论，而不是"7变量摘要拼盘"（每个变量各写一句，互不相干地排排站）。

**解法：不是从7个变量里挑几个，而是固定4个叙事节拍（narrative beats），每个节拍从对应变量里取内容，拼成一条有因果指向的地图，而不是并列清单。**

| 节拍 | 内容来源 | 是否必选 |
|---|---|---|
| A. 你如何感受和给出爱 | V1 | 必选，固定开场 |
| B. 什么会触发/消耗你 | V2 | 必选 |
| C. 关系张力升高时你如何变化 | V3 与 V4 二选一做标题句，规则见下 | 必选（合并计1条） |
| D. 什么帮你真正恢复 + 你现在想往哪练 | V5 + V7 合并成一条收尾、指向 Module 9/10 | 必选 |
| E.（条件性）好事发生时的关系协调模式 | V6 | 仅当 V6 与 V1(CP3) 的交叉验证命中"呼应"或"不一致"时才加入，否则跳过 |

固定 4 条（A/B/C/D）+ 条件性第 5 条（E）= 3-5 条，符合要求，同时保证结构可预测、不会变成清单。

**C 节拍的 V3/V4 选择规则**：
- 若 V3 或 V4 任一方在本用户的答案里触发了"冲突条件"（见各自变量定义），优先选**触发了冲突条件的那一个**作为标题句，另一个降级为补充半句。
- 若两者都触发了冲突条件，只取信号更强的一个（交叉验证支持数更多的优先），另一个的内容留到后面对应的完整模块里展开，不在 Module 1 里堆两条。
- 若两者都没有触发冲突条件（用户答案内部高度一致），默认使用 V4（旗舰变量），因为它信息密度最高，天然适合做"状态画像"的概括句。

**去重规则**：如果 D 节拍要用到的 V5/V7 内容，与 A/B/C 节拍已经引用过的同一批交叉验证数据重叠（例如 V7 的"全卷综合"发现最高杠杆点其实还是落在 V1 的核心需要上），D 节拍必须显式承接前文（"回到你最初的那个核心需要……"），不能当成独立新信息重复陈述。

**输出格式**：每个节拍一句话（不展开论证，论证留给后面对应的完整模块），总长度 4-6 句话的一段紧凑摘要，结尾必须有一句承上启下："后面我们会展开看这几件事各自是什么样子，以及它们之间怎么互相牵动。"——为 Module 9 做铺垫。

## Module 9｜核心张力：生成规则

**要解决的问题**：候选池怎么筛、怎么排序、最多输出几组、怎么避免把正常差异写成"矛盾"。

**候选池**：V1-V7 六字段规范里"冲突条件"字段，取**本次用户答案实际触发的全部条目**（不是罗列全部可能性，只要被真实命中的）。这个池子比 Module 1 用到的更全——Module 1 每个节拍最多浮现1条，Module 9 要看全卷所有被触发的张力，包括没进 Module 1 摘要的那些。

**筛选与排序规则（三步）**：
1. **去重合并**：如果两条被触发的张力本质上是同一个根源模式的不同表现（例如"V1 的 CP1≠CP2"和"V3 的保护策略对外不可见"都源自"你的需要很少被主动说出口"这同一件事），合并成一条更完整的表述，不拆成两条。
2. **按证据强度排序**：优先展示有多个交叉验证共同支持的张力（对应证据语言 Level 3，比单一交叉验证的 Level 2 更值得放在前面）。
3. **按可行动性排序（同强度时的第二排序依据）**：能直接对应一个具体行为改变的张力，优先于纯描述性、用户不知道能做什么的张力。

**输出数量**：最多3组，最少可以是1组甚至0组——**必须允许用户答案内部高度一致、没有值得展开的张力**这种情况存在，不能为了凑数硬造。这条直接继承"报告克制原则"。

**候选池的动态扩展（2026-07-08 压测后补充）**：候选池不能只认各变量预先写死的"冲突条件"字符串。实测发现真实答案里经常出现"3个以上独立数据点方向高度一致，但没有被任何一条预定义条件覆盖"的情况（例如 S6/S7/S2 三题都指向"从不主动说，靠对方或行为自己浮现"，这不属于任何变量定义的冲突条件，但明显是一个值得写的真实模式）。规则补充：允许识别预定义条件之外的模式，但必须满足——①至少3个独立数据点方向一致 ②必须能明确归类为 Corroboration / Contextual Contrast / Complementary Dimensions 三种类型之一，归不进去就不能用。这条同时适用于 Module 1（C节拍的判断）、Module 9（候选池）、Module 10（候选节点）。

**Module 9 与 Module 1 去重规则（2026-07-16 压测后补充，修复"Module 9 重复 Module 1 头条发现"的问题）**：

Module 9 在筛选候选张力前，必须先读取 Module 1 已使用的核心发现，并标记其 root pattern。

如果某个 Module 9 候选张力与 Module 1 已经使用的发现满足以下三项中的**两项**，则视为重复候选，默认排除：
1. 使用相同主变量或相同题目组合；
2. 指向相同心理机制或关系模式；
3. 得出相同结论方向。

重复候选只有在满足"新增价值测试"时才允许保留。**新增价值测试**必须至少满足以下一项：
- 引入了 Module 1 没有展开的新证据；
- 从"描述性发现"推进到"行动性张力"；
- 揭示了一个新的代价、风险或改变节点；
- 与另一个变量形成新的 Contextual Contrast 或 Complementary Dimensions。

如果只是把 Module 1 的结论换一种说法，不得进入 Module 9。**Module 9 的职责不是重复报告最强发现，而是补充用户最值得看见、但还没有被充分展开的 1-3 组核心张力。** 如果排除重复后没有真正有价值的张力，Module 9 允许输出0组，或非常简短地说明："本次答案中未出现需要额外展开的高优先级内部张力。"

**"避免把正常差异写成矛盾"的强制语言规则**：
- 每一条 Module 9 的张力，落笔前必须先归类成 Contextual Contrast 或 Complementary Dimensions 两种类型之一（见 V3_STRUCTURE.md 的 Evidence Network）。**如果一条触发的"冲突条件"没法被归进这两类里的任何一类，就不能作为 Module 9 的内容出现**——说明它更像是数据噪音或者边界模糊的组合，不是真正的结构性张力。
- 禁止词表（本模块尤其高风险，必须显式复查）："矛盾"、"自相矛盾"、"说一套做一套"、"你以为……其实……"、"言行不一"。
- 强制句式：Contextual Contrast 用"在……情况下……，但在……情况下……，这不是矛盾，而是……"；Complementary Dimensions 用"……和……是两件不同的事，共同拼出……"。

## Module 10｜最高杠杆改变点：生成规则

**决策链**：G1（主观成长目标）→ 对照 Module 9 已识别出的张力 → 从候选节点里选一个"最早、最可控、最可能改变后续链条"的点 → 输出**一个**主杠杆点，用完整的 Insight → Choice → Practice → Review 四步展开。

**这是全份报告唯一一个走完整四步的模块**——呼应最初讨论时"不是要求她15个方面全部成长，只找最值得改变的一个节点"。其余模块只到 Insight（+ 简短建议），不做完整的 Practice/Review 展开。

**候选节点筛选的三层过滤（当 Module 9 有多个张力可选时）**：

1. **最早**：候选节点里，优先选在用户自己的状态转换链条上位置最靠前的那个。参考顺序（越靠前越优先）：V2/V3 的 S7（冲突尚未爆发、只是"已经不舒服"阶段）> V3 的 S2/S3（冲突已经开始或信号已经模糊）> V5 的 G6（冲突已经发生，进入修复阶段）。理由：越早期的节点，改变它能防止后面整条链条被激活，杠杆最高。
2. **最可控**：在同样早的候选之间，优先选**完全属于用户自己行为**、不依赖伴侣先改变的节点（呼应单侧数据 Rule 1）。例如"我识别到不舒服后多快说出来"比"我需要伴侣先给我确认"更可控，应该优先。
3. **最可能改变后续链条**：如果前两层还没有筛出唯一节点，选择在全卷交叉验证网络里被**其他变量引用次数最多**的那个（结构上最"上游"，牵动的下游模块最多）。

**与 G1 的对照逻辑（三类，2026-07-16 边界测试后正式扩为三类）**：

| 类型 | 判断标准 | Module 10 写法 |
|---|---|---|
| 一致 | G1 和全卷最高杠杆点指向同一能力 | "你自己的直觉是准的。" |
| 不一致但可补充 | G1 有价值，但全卷显示另一个节点更早、更可控、更高杠杆 | "这个方向值得保留，同时还有一个更早的切入点……" |
| 互补 | G1 和全卷杠杆点不是同一方向，但服务于同一个关系目标（例如"理解TA"与"让自己被理解"，同属"爱如何被感知与传递"这一个更大主题），彼此不冲突 | "这两个方向不是互相替代，而是分别处理关系中的两个侧面。" |

- **一致**：直接针对这个方向给出具体 Practice + Review。
- **不一致但可补充**：不能否定用户的选择，必须先肯定 G1 的方向本身也成立，再补充更早/更可控的起点，两个方向都保留，不强行替换用户的自我判断。
- **互补**：不能说"不一致"，也不能说"一致"——要明确指出两者是同一个更大主题的两个不同侧面（例如"理解TA"和"被TA理解"，同属"关系中的双向理解能力"），分别处理关系中的两个侧面，不构成替代关系。

判断"互补"而不是"不一致"的标准：G1 指向的能力和全卷杠杆点指向的能力，如果分别属于同一个更大关系主题的"给出"侧与"接收"侧（例如表达↔倾听、理解TA↔被TA理解、给空间↔要参与），归为互补；如果两者服务于完全不相关的目的，或者其中一个明显是另一个的前置条件（做不到A就很难做到B），归为"不一致但可补充"。

**输出格式（唯一一处完整展开六要素，2026-07-16 大规模压测后由五要素扩为六要素）**：
0. **g1_alignment**——独立的结构化字段，必须先输出这个判断结果（一致/不一致但可补充/互补），再写下面的内容。压测发现只把这个判断"融进insight散文里"时漏写率约50%，改成必须先输出的独立字段后漏写率降为0，这条纪律不能再退回"融进散文里"的写法。
1. Insight——命名这个模式，文字里自然带出g1_alignment的判断结果："你可能……"
2. Function——它保护了什么、为什么会这样（不评判）
3. Cost——这个模式大概什么时候开始失效
4. Practice——一个具体、可执行的替代动作（不是"多沟通"这种抽象建议）
5. Review——一个 7-14 天内可以自我核对的具体观察任务
