# 报告语气规范

**分工说明**：`REPORT_ANALYSIS_ENGINE.md` 定义"该推断什么、该说什么内容"；本文件定义"这些内容该用什么强度、什么语气、什么句式写出来"。两者都要遵守 `QUESTIONNAIRE_V3_STRUCTURE.md` 里的证据语言四层框架（Level 1-4）和单侧数据硬规则——本文件是把那些原则落成可以直接检查的具体规则和句式库，不是另起一套体系。

这份文档存在的理由：分析逻辑已经基本闭环，现在最大的风险不是"推断错了"，而是**同一套正确逻辑被写成不同强度、不同人格化程度、不同建议颗粒度的报告**——不同模块、不同用户、甚至同一次生成里的不同段落，语气不统一，会让报告读起来像拼凑的。

---

## 1. 证据强度：什么条件下才能说到什么程度

对应 V3_STRUCTURE.md 的 Level 1-4 框架，这里给出可以直接套用的判断规则：

| 触发条件 | 允许的确信程度 |
|---|---|
| 单题作答 | 只能写"可能反映……/更倾向于……"，不能升级成结论 |
| 2 题以上方向一致（Corroboration） | 可以写"你的答案组合显示……" |
| 同一模式在**不同状态情境题**里重复出现（例如 V4 的 G8/S1/S9/G7 这类跨"安全-稳定-忙碌-巨变"状态的题目都指向同一方向） | 才可以升级为"这是你一个相对稳定的关系模式"这个级别的表述——这是比普通 Level 2 更高的门槛，因为它要求信号跨越了不同的情境/状态，不只是同一情境下的多题重复 |
| 跨题推断、结果不确定（Level 4） | 必须用"一种可能的解释是……/也许……"，不得由单题触发，不能写成事实 |

**核心原则**：证据强度只能随数据支持程度往上走，不能因为"这样写更好看/更有洞察力"就跳级。

---

## 2. 措辞等级：三档固定词库

从弱到强，只允许这三档，不能自创中间强度的措辞：

| 档位 | 词库 | 使用条件 |
|---|---|---|
| 弱 | "可能反映……" / "有一种倾向是……" | 单题、Level 1 |
| 中 | "更倾向于……" / "你的答案组合显示……" | 2 题以上方向一致，Level 2 |
| 强 | "在多个情境中持续出现……" / "这是一个相对稳定的模式……" | 跨状态重复验证，见上表第三行 |

**硬性禁止**（任何强度都不能用这些句式，因为它们把"倾向"偷换成"本质"）：
- "你就是……"
- "你一直……"
- "你的本质是……"
- 任何把单次问卷作答等同于人格定论的句式

**措辞多样性（2026-07-08 压测后补充）**：同一个比喻或意象（例如"这条线索""这条线"）在同一份报告全篇最多出现2次。压测发现模型容易在 Module 1 和 Module 10 之间反复使用同一个比喻词收尾，读起来像偷懒而不是呼应。如果 Module 10 需要承接 Module 1 提到的模式，用换一种说法复述内容本身，而不是重复同一个比喻词。

---

## 3. 共情比例：不是固定百分比，是固定顺序

不规定"30% 共情、70% 分析"这种比例，而是规定**每一段涉及用户某个模式的内容，必须按这个顺序展开**：

1. **先准确描述**——这个模式具体是什么，不加评价
2. **再解释功能**——它为什么合理、保护了什么（这一步就是"共情"发生的地方——共情来自"我理解你为什么会这样"，不是空洞的安慰话术）
3. **最后指出代价**——这个模式在什么情况下会开始失效

这个顺序本身就是问卷设计阶段"功能+代价"选项文案原则（S3/S6 等构念用过的写法）在报告生成阶段的延续——问卷选项怎么写代价，报告怎么写代价，是同一套逻辑在两个环节的应用。

**禁止**：跳过"解释功能"直接从"描述"跳到"代价"，那样会读起来像指责；也禁止只描述+共情、不指出代价，那样会变成没有信息量的安慰。

---

## 4. 挑战强度：三个条件同时满足才能写

"挑战"（指出用户模式里可能存在的问题）必须同时满足：

1. **有跨题证据**——不能凭单题就发起挑战
2. **指向可改变的行为**——不能挑战一个用户无法改变的东西（比如天生倾向、无法控制的处境）
3. **不归因人格缺陷**——挑战的是模式和它的后果，不是给人贴标签

**范例（可以写）**：
> "你希望伴侣理解你的需求，但在不舒服的早期阶段又倾向于先压下去。这可能让对方接收到你的失望信号的时间，晚于你自己感受到失望的时间。"

这条挑战有跨题证据（V1的需求+V3的S7早期策略）、指向可改变的行为（表达时机）、没有说"你不会沟通"这种人格化的话。

**反例（不能写）**：
> "你不会好好沟通，所以关系才会出问题。"

---

## 5. 建议颗粒度：除 Module 10 外都只给一个提醒

| 模块 | 建议颗粒度 |
|---|---|
| Module 10（最高杠杆改变点） | 唯一一处完整展开 Insight → Function → Cost → Practice → Review 五步 |
| 其他所有模块 | 最多"一个提醒 / 一个观察角度"，不展开具体训练计划，不给多步骤行动方案 |

这条防止报告每一段读起来都像作业清单。用户读完大部分模块应该是"哦，原来是这样"，只有读到 Module 10 才进入"接下来我可以这样做"的具体执行状态。

## 6. 禁止表达：六个语义族（2026-07-08 扩展为语义级规则，不是字符串黑名单）

**核心原则**：下面每一类给的词表**只是例子，不是穷举**。词表永远列不完——模型会用近义词绕过一个固定黑名单。真正要执行的是每一类后面的**语义测试**：写完一句话，问自己这句话是否落入这个测试描述的情况，落入就要重写，不管有没有用到词表里那个"标准词"。

### ① 病理化
把模式描述成疾病、缺陷、需要被治愈的异常状态。
**例子**（含常见绕路说法）："你有创伤"、"你缺爱"、"你不健康"、"情感匮乏"、"情绪依赖严重"、"情感需求过度"、"你的情感账户是空的"。
**语义测试**：这句话是否暗示用户的状态是"缺陷/异常/需要被修复的"，而不是"一种有功能、有代价的权衡"？是，就违规。
**替代写法**：换成"功能+代价"框架——不写"你情感匮乏"，写"你更倾向于从多个来源获得情感支持，不只依赖伴侣一个人"。

### ② 类型化
把用户归入一个流行心理学/依恋类型标签，暗示这是固定、可诊断的身份类别。
**例子**："焦虑型"、"回避型"、"焦虑型依恋"、"回避型依恋"、"高度依恋焦虑"、"亲密恐惧"、"不健康的依恋模式"、"矛盾型依恋"、"混乱型依恋"、"讨好型人格"。
**语义测试**：这句话能不能被套进"你是____型"的句式而意思不变？能，就违规——问卷不是依恋类型测评，任何类型归类都是编造。
**替代写法**：只描述行为本身和触发情境，不给它起一个类型名字。

### ③ 宿命化
把倾向描述成永久、不可改变、命中注定。
**例子**："你注定……"、"你总会……"、"你永远都会……"、"这就是你的宿命"、"你天生就是这样"、"你这辈子都……"。
**语义测试**：这句话是否暗示这个模式没法通过练习或情境改变？是，就违规——报告的立场是"模式可以被看见、被调整"，宿命化直接和产品定位冲突。
**替代写法**：用"倾向于/更容易/目前的模式是"这类可变化的措辞。

### ④ 来源臆测
给行为模式编造一个未经证实的心理成因（童年、创伤经历、原生家庭等问卷根本没采集的信息）。
**例子**："这说明你童年经历了……"、"这可能源于你的原生家庭……"、"这是一种关系创伤反应"、"这是害怕被抛弃的表现"、"这是缺乏安全感留下的痕迹"。
**语义测试**：这句话是否在为一个模式指定具体心理成因，而这个成因不在15题回答范围内？是，就违规——即使听起来很有洞察力，也是编造。
**替代写法**：只停在"这是什么模式、它保护了什么、代价是什么"，不猜"为什么会这样"。

### ⑤ 伴侣诊断
通过用户回答反推、断言伴侣本人的心理状态或人格（违反单侧数据 Rule 3）。
**例子**："TA是回避型"、"TA缺乏共情能力"、"TA控制欲强"、"TA是控制型人格"、"TA有边界问题"、"TA不成熟"。
**语义测试**：这句话的主语是不是"TA"，而且是在做心理定性（不是复述用户对某个具体事件的描述）？是，就违规。
**替代写法**：单侧数据条件句——"如果伴侣也倾向于……"。

### ⑥ 道德评价
把某种模式描述成比其他模式更成熟、更健康、更值得推崇，形成道德位阶。
**例子**："成熟的人应该……"、"健康关系一定……"、"更高情商的做法是……"、"真正爱一个人就会……"、"不健康的依恋模式"（这个词同时踩了②类型化和⑥道德评价两条线）。
**语义测试**：这句话是否暗示存在一种"正确/更高级"的关系方式，让用户目前的模式显得"低一等"？是，就违规——7个变量、报告里的所有模式都是平行的真实权衡，不是能力测验。
**替代写法**：只描述功能和代价，不排名。

**跨类提醒**：不少常见说法会同时踩中好几类（"不健康的依恋模式"同时是类型化+道德评价，"关系创伤反应"同时是病理化+来源臆测）——只要踩中任意一类的语义测试就必须改写，不需要先判断它具体属于哪一类。

这六类是全局禁止规则，适用于全部15题、全部7个变量、全部10个模块。各变量在 `QUESTIONNAIRE_OPTIONS.md` 里另有一些构念特定的"不得推断"补充规则（比如"G6选'先恢复亲密感'不得解释为缺乏安全感"），那些是这六类的具体案例，不是另一套独立标准。

## 7. 单侧数据条件句：固定句式

对应 V3_STRUCTURE.md 的单侧数据硬规则，这里给出可以直接使用的句式库：

**互动循环推演的标准句式**（二选一，根据语境选更顺的）：
> "如果伴侣也倾向于 X，你们之间可能形成 Y。"

> "在某些关系中，如果对方的策略恰好与这一模式相互强化，可能出现 Y 这样的循环。"

**首次出现互动循环推演时，必须紧跟这句固定声明**（不是页面角落的小字免责声明，是正文的一部分）：
> "以下互动推演基于你的单侧回答，用于帮助识别可能模式，不代表对伴侣或双方关系的判断。"

这句声明只需要在报告里**第一次**出现互动循环推演时说一次，不必每次重复，但必须出现在第一次，且要在正文里，不能只放在页面底部的通用免责声明区。

## 违规自查清单（供 Prompt 自检 / 人工 review / 模型评测直接使用）

生成或审阅任意一段报告文本时，逐条确认：

- [ ] 这段话的确信程度，和支持它的证据数量（单题/2题以上/跨状态重复）匹配吗？
- [ ] 用词是否落在"可能反映 / 更倾向于 / 在多个情境中持续出现"这三档之内，没有用"你就是/你一直/你的本质是"？
- [ ] 涉及用户模式的段落，是否按"先描述、再解释功能、最后指出代价"的顺序写的？
- [ ] 如果这段话是"挑战"性质，是否同时具备跨题证据、指向可改变行为、不归因人格缺陷这三点？
- [ ] 这个模块除了 Module 10，是否只给了一个提醒/观察角度，没有展开成训练计划？
- [ ] 有没有出现病理化、类型化、宿命化、来源臆测、伴侣诊断、道德评价这六类禁止表达——不只是查有没有出现词表里的标准词，是逐句过一遍六个语义测试？
- [ ] 如果出现了互动循环推演，是否用了固定条件句式，并且是本次报告里第一次出现互动循环推演时带上了那句声明？
