# MVP 产品范围定义

## 产品定位
面向亲密关系中的个人用户的 AI 自我探索产品。用户完成 15 道场景化选择题，
获得一份个性化的"亲密关系说明书"——不是人格分类，不是诊断，
而是帮用户理解"原来我是这样的人，原来我需要的是这个"。

## 四项关键决策

| 决策 | 选择 | 核心逻辑 |
|---|---|---|
| 商业模式 | 完全免费 | 现阶段只验证"报告是否被信任、被喜欢"，不引入支付这类高风险高复杂度环节 |
| 账号体系 | 不设登录，专属链接 + 可选邮箱 | 零门槛使用，但报告可保存、可分享，邮箱作为未来触达的种子资产 |
| 报告呈现 | 网页展示（移动端优先）+ 一键生成分享图 | 匹配小红书"晒图"传播习惯，是成本最低的自然增长杠杆 |
| 问卷对象 | 纯个人自测 + 报告页"邀请 TA 也来测"引导位 | 忠于产品定位，同时为未来双人功能埋自然的传播钩子 |

## MVP 功能范围

**包含：**
1. 首页（简单介绍 + "开始测试"按钮）
2. 15 题问卷页（ABCDE 选择 + 部分题 F 自由输入，移动端优先）
3. 提交后调用 AI 生成个性化报告
4. 报告展示页（专属链接访问，10 个模块内容 + "邀请 TA 也来测"入口 + 一键生成分享图 + 可选"留邮箱保存"）
5. 问卷答案与生成报告存库，方便查看数据、分析报告质量

**不包含（明确排除）：**
- 用户注册 / 登录 / 密码
- 任何形式的支付
- 双人对照 / 情侣匹配功能
- "我的测试历史"页面
- 后台管理系统（直接看 Supabase 数据表）
- 多语言支持

## 技术架构

- **前端**：Next.js
- **部署**：Vercel（自动发布、可绑定自定义域名）
- **数据库**：Supabase（Postgres，带可视化界面）
- **AI 生成层**：Provider 可切换（Claude API / OpenAI API），通过统一的
  "报告生成"接口调用，具体使用哪个 Provider 由配置项决定，不绑定单一厂商。
  切换或对比模型时只改配置，不改问卷页/报告页代码。
- **安全**：所有 API Key 只存在于 Vercel 服务器端环境变量中，浏览器端代码
  永远不会包含任何密钥。

## 数据结构（Supabase `reports` 表）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | uuid（主键） | 数据库内部唯一标识 |
| report_id | text（唯一索引） | 对外专属链接用的短标识，如 `x7f9k2`，用于生成 `/report/x7f9k2` |
| email | text（可空） | 用户可选填写的邮箱，用于保存/未来触达 |
| answers | jsonb | 15 题的结构化选择记录，如 `{"q1":"B","q2":"A",...}` |
| f_text_inputs | jsonb | 用户选 F 选项后填写的自由文本，按题号存，如 `{"q3":"..."}` |
| full_report | jsonb | AI 生成的完整报告内容（按模块结构化存储） |
| summary | text | 报告摘要/金句，用于分享图和简短展示 |
| model_used | text | 记录实际调用的模型，如 `claude-sonnet-5` / `gpt-4o`，用于对比不同模型质量 |
| prompt_version | text | 记录使用的 Prompt 模板版本号，便于后续迭代时追溯效果差异 |
| created_at | timestamptz | 自动记录创建时间 |

这个结构从第一版就把"未来做数据分析、对比模型效果、迭代 Prompt"所需的字段
都留好了，避免后期还要改表结构、迁移旧数据。
