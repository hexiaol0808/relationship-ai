# REPORT_INPUT_TEMPLATE_V1

配合 REPORT_SYSTEM_PROMPT_V1.md 使用。系统 Prompt 定义"规则"，本文件定义"每次调用时，用户轮该长什么样"。

---

## 1. 数据库原始存储格式（compact，字母级）

对应 MVP_SCOPE.md 的 `answers` / `f_text_inputs` 字段，这是问卷提交后落库的样子：

```json
// answers
{
  "q1": {"first": "A", "second": "B"},
  "q2": {"first": "A", "second": "B"},
  "q3": {"value": "C"},
  "q4": {"first": "A", "second": "C"},
  "q5": {"value": "A"},
  "q6": {"first": "A", "second": "D"},
  "q7": {"value": "A"},
  "q8": {"value": "A"},
  "q9": {"value": "B"},
  "q10": {"value": "A"},
  "q11": {"value": "B"},
  "q12": {"value": "A"},
  "q13": {"value": "A"},
  "q14": {"value": "D"},
  "q15": {"value": "A"}
}

// f_text_inputs（只有选了 F 的题目才有 key，本例没有人选 F，所以是空对象）
{}
```

这个格式**不能直接喂给模型**——模型不知道"q1.first=A"对应哪句话。发送给模型之前，应用层必须先用 `QUESTIONNAIRE_OPTIONS.md` 做一次"字母→完整文字"的展开，规则见下。

---

## 2. 展开规则（应用层职责，不是模型的职责）

对每一题：
1. 取出题目标题（不含分析标签）。
2. 取出用户选中字母对应的选项完整文字（标题+描述，如果该题选项有短标题的话）。
3. 如果是选二/两步类型，分别标注"第一优先/第二优先"或"第一步/第二步（相对能适应）"。
4. 如果选了 F，附上 `f_text_inputs` 里对应的自由文本；如果为空，标注"（用户未提供具体说明）"。
5. 每题标注构念代码（对照表见 REPORT_SYSTEM_PROMPT_V1.md 层2）。

**单题渲染模板**：

```
Q{n}（{construct_code}）｜{题目标题}
选择：{完整选项文字}
```

选二/两步类型：

```
Q{n}（{construct_code}）｜{题目标题}
第一优先：{完整选项文字}
第二优先：{完整选项文字}
```

## 3. 完整示例（一组虚构测试答案，可直接用于压测）

```
Q1（CP1）｜哪两种方式最能让你感受到爱？
第一优先：高质量陪伴——TA会主动留出一段专注、不被手机或工作打断的时间，认真和你聊天，或者陪你一起做一件事。
第二优先：肯定的话语——TA会主动把欣赏、感谢、在乎或者想念说出来，让你明确知道自己的重要性。

Q2（CP2）｜你通常怎样表达自己的爱？
第一优先：服务的行动——看到TA遇到麻烦、压力或者现实困难时，我会自然地想帮忙处理、解决或者分担。
第二优先：身体接触——见面时，我会自然地想拥抱、牵手、亲吻或者靠近TA。

Q3（S6）｜当爱没有对上频道
选择：我会直接告诉TA我的需求——我会直接告诉TA，也愿意给时间调整；如果需要长期反复提醒，我也可能感到疲惫。

Q4（CP3）｜关系中，什么会越来越消耗你？
第一步：TA始终很难真正理解我的内心世界——我常常有一种"说了也没用，TA不会真正懂我"的感觉。
第二步（相对能适应）：TA有自己的生活和未来，我总不确定自己在其中的位置——我会越来越怀疑："TA真的选择了我吗？"

Q5（S5）｜以下哪一种行为最容易立刻引发你的情绪？
选择：我认真表达感受时，TA立刻否定、淡化，或者说我"想太多""太敏感"。

Q6（G7）｜现实发生巨大变化时，你相对最能接受关系失去哪一部分？
第一优先：日常互动减少——联系和生活分享明显减少，但彼此仍保持基本连接，不会无故失联。
第二优先：生活融合程度下降——彼此不再深入参与对方的大部分日常安排，但重要变化和重大决定仍会主动同步、共同商量。

Q7（G8）｜当你开始越来越信任TA后，你最明显的变化是什么？
选择：表达真实情绪——不再过度隐藏自己，更愿意直接表达真实情绪和想法。

Q8（S1）｜关系稳定时，你最舒服的相处状态是什么？
选择：我喜欢彼此自然参与很多日常：经常分享小事、讨论安排，也比较清楚对方最近的生活状态。

Q9（S9）｜双方都很忙时，以下哪种状态最接近你的真实需求？
选择：每天至少有一次明确的联系就够了，不一定聊很久，但我需要感觉到我们每天都有碰到彼此。

Q10（S2）｜当关系突然有点不对劲时，你最先会怎么做？
选择：我会找机会直接问TA最近怎么了，希望尽快弄清发生了什么。

Q11（S7）｜还没正式吵起来，但你已经感到不舒服了，你通常怎么处理？
选择：不舒服之后，我通常比较快就会说出来。

Q12（S3）｜冲突刚发生时，你最先想处理什么？
选择：尽快把事情讲清楚——我想马上继续沟通，不喜欢问题悬在那里，希望尽快知道我们到底怎么回事。

Q13（G6）｜一次冲突之后，哪一种修复最能让你恢复对关系的信任和连接？
选择：先获得真正的理解——TA认真听懂了我为什么难过，也能准确理解这件事对我的意义。即使我们暂时还没有找到完整的解决办法，我也会开始感觉好多了。

Q14（G2）｜当TA迎来重要的发展机会时，以下哪一件事是你最优先希望发生的？
选择：确认我仍然在TA的未来里——我最需要知道，这个变化发生之后，TA依然把我们的关系和共同未来放在认真考虑的位置。

Q15（G1）｜如果只能提升一种关系能力，你最想练习什么？
选择：表达能力——能够更直接、清楚地表达自己的感受和需要，不指责、不试探、不积累不满。
```

**这组示例预期会命中的信号**（供压测时核对模型有没有算对，不要把这段发给模型）：
- V1 应触发张力：CP1 首选（高质量陪伴）≠ CP2 首选（服务的行动）
- V2 应显示 Corroboration：CP3 首选机制（不被理解）与 S5（情绪被否定/说太敏感）同源
- V4 应触发张力：S1 显示高融合理想，但 G7 显示对"生活融合度下降"也能接受——"理想中偏好紧密关系，但现实适应力其实不差"
- V5：S3（尽快讲清楚）与 G6（先获得真正理解）方向基本一致，**不应该**被判定为张力——这是用来检验模型会不会"为了凑数硬造张力"的反例

这组答案属于"跨状态有一定张力但不极端"的类型，建议后续再单独构造"高度一致型"和"表面矛盾实为情境对照型"两组，三组一起压测（详见对话中约定的三类测试样本）。
